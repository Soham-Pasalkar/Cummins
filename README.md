# PUMA Analytics Pro

**Production-Grade Test Data Validation & Analysis Platform for Cummins Engineering**

## Overview

PUMA Analytics Pro is a Streamlit-based application designed specifically for Cummins testing engineers to analyze, validate, and visualize large-scale sensor data generated by PUMA software. The platform handles 600-1800 sensor columns efficiently with support for both transient and steady-state tests.

## Features

### Core Capabilities
- ✅ **Large-Scale Data Handling**: Efficiently process 600-1800 sensor columns
- ✅ **Multi-Test Support**: Transient (1Hz, 3Hz, 10Hz) and Steady-State tests
- ✅ **Advanced Validation**: Threshold, Rate-of-Change, Duration-based rules
- ✅ **Baseline Comparison**: Delta analysis against reference tests
- ✅ **Category-Based Analysis**: Automatic sensor categorization
- ✅ **Multi-Cycle Segmentation**: Analyze transient cycles individually
- ✅ **Interactive Visualizations**: Plotly-powered charts and heatmaps
- ✅ **Export Capabilities**: JSON, CSV, Excel report generation

### Validation Rules
1. **Threshold Validation**: Min/Max bounds checking
2. **Rate of Change (ROC)**: Detect rapid parameter changes
3. **Duration Filter**: Ignore transient spikes below minimum duration
4. **Statistical Outliers**: Z-score based anomaly detection
5. **Missing Data Detection**: Track data quality issues

### Test Type Detection
The application automatically detects:
- Test type (Transient vs Steady State)
- Sampling frequency (1Hz, 3Hz, 10Hz)
- Test duration
- Total sensor count

## Installation

## Usage Guide

### 1. Data Preparation

#### Test Data File (Required)
- **Format**: Excel (.xlsx or .xls)
- **Structure**: 
  - First column: Time/Timestamp (in seconds)
  - Remaining columns: Sensor data (600-1800 columns)
- **Example columns**: `Time`, `EngineSpeed_rpm`, `CoolantTemp_C`, `ExhaustPress_kPa`, etc.

#### Configuration File (Required)
- **Format**: Excel (.xlsx or .xls)
- **Required columns**:
  - `Parameter`: Sensor name (must match test data columns)
  - `Min`: Minimum allowed value (optional)
  - `Max`: Maximum allowed value (optional)
  - `Unit`: Unit of measurement (optional)
  - `ROC`: Rate of change limit (optional)
  - `MinDuration`: Minimum violation duration in seconds (optional)

#### Baseline File (Optional)
- **Format**: Excel (.xlsx or .xls)
- **Structure**: Same as test data file
- **Purpose**: Compare current test against baseline for delta analysis

### 2. Running Analysis

1. **Upload Files**
   - Click "Test Data (Excel)" to upload your PUMA test file
   - Click "Configuration (Excel)" to upload validation rules
   - (Optional) Click "Baseline Data" to upload reference test

2. **Select Analysis Mode**
   - **Standard Validation**: Basic validation with all sensors
   - **Multi-Cycle Analysis**: Segment transient tests into cycles
   - **Comparison Mode**: Enhanced baseline comparison

3. **Click "🚀 Run Analysis"**
   - Progress bar shows validation status
   - Results display automatically upon completion

### 3. Interpreting Results

#### Test Information Panel
- **Test Type**: Automatically detected (Transient/Steady State)
- **Frequency**: Sampling rate (1Hz, 3Hz, 10Hz)
- **Duration**: Total test time in seconds
- **Sensors**: Number of parameters monitored
- **Data Points**: Total measurements collected

#### Validation Dashboard
- **Total Parameters**: Number of sensors validated
- **PASS**: Parameters within specification
- **WARNING**: Minor violations detected
- **FAIL**: Critical violations or missing data
- **Avg Violation %**: Overall test quality metric

#### Category Breakdown
Sensors automatically grouped into categories:
- Temperature
- Pressure
- Speed
- Torque/Load
- Flow
- Electrical
- Emissions
- Other

#### Status Heatmap
- **Green**: PASS - All measurements within spec
- **Yellow**: WARNING - Minor violations detected
- **Red**: FAIL - Critical issues found

### 4. Advanced Features

#### Multi-Sensor Time Series
- Select up to 10 sensors simultaneously
- Overlay baseline data for comparison
- Synchronized zoom and pan across plots

#### Parameter Drilldown
- Detailed statistics (mean, std, min, max)
- Baseline delta calculation
- Time series visualization with baseline overlay

#### Export Options
- **JSON**: Complete session data with metadata
- **CSV**: Summary table for external analysis
- **Excel**: Multi-sheet report with category statistics
- **Failed Only**: Quick export of problem parameters

## Best Practices

### Data Quality
- Ensure time column is in seconds
- Remove header rows (except column names)
- Use consistent column naming
- Handle missing values appropriately

### Configuration Setup
- Start with wide tolerances, then refine
- Use MinDuration to filter transient spikes
- Set ROC limits for critical fast-changing parameters
- Group related parameters for easier analysis

### Performance Optimization
- For very large files (>1GB), consider data decimation
- Use Multi-Cycle Analysis for long transient tests
- Filter to specific categories when analyzing subsets
- Export results regularly to avoid re-computation

### Validation Strategy
1. Run Standard Validation first to identify failures
2. Review Category Breakdown to spot systematic issues
3. Use Parameter Drilldown for root cause analysis
4. Compare against Baseline to detect test-to-test variations
5. Export failed parameters for engineering review

## Troubleshooting

### Common Issues

**Issue**: "Time column not found"
- **Solution**: Ensure first column is named "Time", "Timestamp", or "t"

**Issue**: Parameters not appearing in results
- **Solution**: Verify parameter names in config match test data exactly

**Issue**: High memory usage
- **Solution**: Close other applications, use data decimation, or upgrade RAM

**Issue**: Slow validation
- **Solution**: Reduce number of parameters in config or use category filtering

**Issue**: Export fails
- **Solution**: Check file permissions, ensure sufficient disk space

## Advanced Configuration

### Custom Categorization
Edit the `_categorize_parameter` method to customize sensor categories based on your naming conventions:

```python
def _categorize_parameter(self, param_name: str) -> str:
    param_lower = param_name.lower()
    
    # Add custom categories
    if any(x in param_lower for x in ['boost', 'turbo']):
        return "Boost System"
    elif any(x in param_lower for x in ['fuel', 'injection']):
        return "Fuel System"
    # ... add more categories
```

```python
@staticmethod
def custom_rule(values, params):
    # Implement custom validation
    violations = np.zeros(len(values), dtype=bool)
    # Your logic here
    return violations
```
